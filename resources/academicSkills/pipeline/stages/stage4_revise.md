# Stage 4: REVISE

## Purpose
Address review feedback and improve draft

## Agents
- `revision_coordinator` — Manages revision process
- `content_improver` — Improves clarity and flow
- `citation_fixer` — Resolves citation issues

## Process
1. Revision coordinator analyzes review feedback
2. Content improver addresses clarity and coherence issues
3. Citation fixer resolves citation problems
4. Revised draft is generated
5. Revision notes document changes

## Inputs
- First Draft (from Stage 2)
- Review Report (from Stage 3)
- Improvement Suggestions (from Stage 3)

## Outputs
- Revised Draft
- Revision Notes (what changed and why)
- Updated Citation List

## Quality Gates
- [ ] All review comments addressed
- [ ] No new issues introduced
- [ ] Citations remain accurate
- [ ] Score improved (if measurable)

## User Checkpoint
User confirms:
1. All critical issues resolved
2. Changes are appropriate
3. Ready for finalization

## Revision Process
1. Read review feedback carefully
2. Prioritize by severity (High > Medium > Low)
3. Address each issue systematically
4. Document changes made
5. Verify no new issues introduced

## Mode Differences
- **Academic Mode**: Address all feedback, document changes
- **Engineering Mode**: Focus on critical issues only
